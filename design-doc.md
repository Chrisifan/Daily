# 智能工作台桌面应用设计文档

## 1. 产品定位

智能工作台是一款面向知识工作者的桌面应用，围绕“当天该做什么、现在最该关注什么、不同工作区如何切换”三个核心问题，提供统一入口。

产品目标：

- 把天气、日程、工作区、邮件、日历、待办等分散信息收敛到一个可操作的桌面工作台。
- 降低用户在多个应用之间切换的成本。
- 通过智能聚合与提醒能力，帮助用户更快进入工作状态。

目标用户：

- 需要同时处理多个项目或身份的个人工作者
- 需要在邮件、会议、任务和文档之间频繁切换的团队成员
- 对效率工具、工作台、自定义工作流有较强需求的用户

## 2. 核心设计原则

- 当日优先：默认先展示“今天”和“现在”，而不是把所有信息平铺出来。
- 智能但克制：智能能力为决策和组织服务，不打断主流程。
- 工作区清晰：所有信息都可以落到具体工作区，避免上下文混乱。
- 系统融合：逐步接入系统日历、邮件和本地能力，而不是把应用做成信息孤岛。
- 桌面沉浸感：界面更像一个“工作场”，而不是网页式后台。

## 3. 信息架构

### 3.1 一级结构

1. 首页概览
2. 日程总览
3. 工作区
4. 收件与通知
5. 智能助手
6. 设置与集成

### 3.2 首页概览包含内容

- 与当前位置天气相关的动态背景
- 今日日期、当前时段、天气摘要
- 当日日程时间轴
- 高优先级事项提醒
- 工作区快捷入口
- 今日收件摘要
- 日历冲突或空档提示

### 3.3 工作区包含内容

- 已有工作区列表
- 新建工作区
- 工作区状态概览
- 与工作区绑定的项目、文件、日程、邮件与快捷动作

## 4. 关键功能拓展

### 4.1 首页概览

在你提出的基础上，首页建议不仅展示“今天的日程卡片”，还应提供以下增强能力：

- 今日时间轴：按小时展示会议、专注时段、截止时间与提醒
- 重要性分层：使用优先级和影响范围区分普通事项、关键事项、风险事项
- 当前焦点模块：只突出“接下来 90 分钟最重要的事”
- 日程总览入口：支持切换为日、周、月视图
- 空档推荐：当检测到时间空档时，推荐处理待办、回信或进入深度工作

### 4.2 天气感知界面

首页背景不只是装饰，建议让天气与工作节奏联动：

- 晴天：光感强、对比清晰、节奏轻快
- 阴天：柔和雾面材质、降低视觉刺激
- 雨天：动态雨幕、玻璃质感、强调安静专注
- 夜晚：低照度背景、增强当前任务聚焦
- 极端天气：加入提示，例如出行会议提醒、通勤建议

天气信息可驱动：

- 背景动效
- 色板切换
- 出行类日程提示
- 智能建议文案

### 4.3 日程系统

建议建立统一的“事件模型”，让不同来源的内容都能转成标准日程：

- 基础字段：标题、开始时间、结束时间、重要性、来源、所属工作区、地点、备注
- 高级字段：是否可移动、是否需要准备材料、是否与他人协作、是否与邮件关联

日程能力应包括：

- 手动创建、编辑、拖拽调整
- 按来源过滤：系统日历、邮件提取、手动创建、工作区计划
- 冲突检测：会议时间重叠、准备时间不足、通勤风险
- 智能补全：从会议标题、邮件正文中自动识别地点、参与者、材料
- 一键转任务：把日程中的行动项转为待办

### 4.4 邮件与系统日历集成

邮件和日历是首页信息的核心输入源，建议以“聚合、理解、建议”为主线接入，而不是第一版就直接控制外部系统。

邮件接入分阶段方案：

第一阶段：

- 接入 Gmail、Outlook Mail 或 IMAP 邮箱
- 拉取未读邮件、今日邮件、带附件邮件
- 生成发件人、主题、摘要、时间等基础卡片
- 在首页展示与今日任务相关的邮件摘要

第二阶段：

- 从邮件中提取会议时间、地点、截止时间、行动项
- 自动识别邮件更适合归属哪个工作区类型
- 对可执行邮件生成建议动作，例如“加入日程”“加入待办”“归档到工作区”

第三阶段：

- 邮件线程与工作区详情双向绑定
- 检测会议改期、主题变化、关键附件更新
- 对高优先级来信自动提升提醒等级

邮件可接入来源：

- Gmail API
- Microsoft Graph Mail API
- IMAP 邮箱

日历接入分阶段方案：

第一阶段：

- 接入本地或云端日历做只读同步
- 读取今日 / 本周事件
- 在首页展示当天安排与冲突提示

第二阶段：

- 识别日历事件对应的工作区归属
- 从邮件与工作区上下文补全事件信息
- 识别准备时间、通勤时间和专注时间建议

第三阶段：

- 会议改期检测
- 重叠冲突识别
- 日程与待办、邮件、工作区联动

日历可接入来源：

- macOS 系统日历
- Google Calendar
- Outlook Calendar
- iCloud Calendar

需要注意：

- 第一版优先做只读同步和智能建议
- 不建议早期自动写回系统日历
- 所有外部接入都要有明确权限提示、同步状态提示和断连处理

### 4.5 工作区系统

工作区不是普通文件夹，而是信息上下文容器。首页上的日程、邮件、文件、待办都应该尽可能能归属于某个工作区。

工作区通用能力：

- 有名称、类型、颜色、状态和当前目标
- 能绑定邮件、日历、文件夹、任务和联系人
- 能展示最近活动、当前重点和下一步建议
- 能通过模板快速创建

工作区视图建议具备：

- 列表视图：快速浏览所有工作区
- 看板视图：按状态区分活跃、待跟进、归档
- 详情页：显示该工作区的时间线、相关邮件、待办、会议和资料
- 智能摘要：自动概括本周进展、风险和下一步动作

### 4.6 工作区类型模板

根据当前需求，建议把工作区预设为四类：

#### 4.6.1 代码工作区

适合内容：

- 开发任务
- bug 修复
- code review
- API 联调
- 技术会议

默认模块：

- 今日编码任务
- Git 分支 / PR / 提交摘要
- 相关 issue
- 技术邮件
- 会议安排
- 本地项目入口

智能建议：

- 从邮件识别 review 请求、bug 反馈、上线通知
- 从日历识别 standup、评审、发版会议
- 汇总今天改动的模块和阻塞项

#### 4.6.2 图片处理工作区

适合内容：

- 设计稿处理
- 图片修图
- 素材整理
- 批量导出
- 视觉评审

默认模块：

- 当前素材集
- 导出任务
- 图片规格
- 附件邮件
- 设计评审日程
- 最近文件

智能建议：

- 从邮件提取附件和修改要求
- 根据日历安排生成会前导出清单
- 跟踪版本变化和交付状态

#### 4.6.3 写作工作区

适合内容：

- 文章
- 文案
- 提纲
- 周报
- 调研整理

默认模块：

- 当前稿件
- 提纲结构
- 截止时间
- 参考资料
- 修改意见邮件
- 采访或讨论安排

智能建议：

- 从邮件提取修改意见
- 自动生成今日写作计划
- 自动总结本周写作进度

#### 4.6.4 通用工作区

适合内容：

- 临时项目
- 行政事务
- 个人安排
- 跨类型任务

默认模块：

- 今日事项
- 日历安排
- 收件摘要
- 快捷入口

### 4.7 工作区自动归类规则

为了减少手动整理成本，建议加入轻量级工作区分类策略：

- 邮件标题或正文包含 PR、commit、bug、接口、发版
  默认归类为代码工作区
- 邮件包含 PSD、海报、导出、尺寸、图片、素材
  默认归类为图片处理工作区
- 邮件包含提纲、文章、润色、文案、周报、稿件
  默认归类为写作工作区
- 无法准确判断时
  进入通用工作区或等待用户确认

### 4.8 收件与通知中心

为了让首页不被打扰，建议把通知集中管理：

- 邮件更新
- 日历变更
- 工作区提醒
- 智能建议
- 系统级事件，例如网络异常或同步失败

通知支持：

- 仅提醒高优先级
- 批量稍后处理
- 直接转成任务或加入某个工作区

### 4.9 智能助手

智能工作台的“智能”部分建议围绕具体工作流，而不是做一个泛聊天入口。

可落地能力：

- 今天帮我排一下优先级
- 根据邮件内容生成日程建议
- 帮我把本周会议整理成工作区摘要
- 检测今天是否存在冲突和空档
- 根据当前天气和会议地点给出出发建议
- 为某个工作区生成周报摘要

## 5. 首页 UI 方案

### 5.1 布局建议

- 顶部：轻量品牌区与全局操作
- 中央：一张主卡展示当前时间、天气和核心概览入口
- 右侧：四张小卡展示日程、工作区、邮件、日历摘要
- 底部：一张今日待办卡和一张今日总览 / 工作区卡

### 5.2 首页首屏模块

1. 天气驱动背景
2. 当前时间与问候主卡
3. 小型摘要卡：日程 / 工作区 / 邮件 / 日历
4. 今日待办卡
5. 总览与工作区卡

### 5.3 视觉风格建议

- 整体风格：偏桌面原生、冷静、沉浸、轻玻璃材质
- 天气背景：大面积氛围层配合少量粒子或流动效果
- 颜色策略：主色不固定，由天气与时段驱动
- 字体策略：标题有识别度，正文偏工具感和可读性

## 6. 关键页面建议

### 6.1 日程总览

- 日 / 周 / 月切换
- 可按工作区、来源、重要性过滤
- 可查看冲突、准备时间和关联邮件

### 6.2 工作区列表

- 展示名称、状态、关联任务数、最近活动、负责人或联系人
- 支持固定常用工作区
- 支持快速创建新工作区模板

### 6.3 工作区详情

- 顶部显示目标与状态
- 中部展示时间线、待办、会议与资料
- 右侧展示智能摘要、风险与建议动作

### 6.4 集成设置页

- 邮箱账户绑定
- 日历账户绑定
- 系统权限状态
- 同步频率设置
- 断连与错误日志

## 8. 集成架构与同步策略

### 8.1 集成层结构

建议采用统一接入层，把不同服务的差异屏蔽掉：

- Connector 层
  负责调用 Gmail、Graph、Google Calendar、系统日历等外部接口
- Normalizer 层
  把外部数据转成统一的 InboxItem 和 ScheduleItem
- Router 层
  根据内容把数据归属到工作区类型或具体工作区
- Suggestion 层
  生成“加入待办”“加入日程”“绑定工作区”等建议动作

### 8.2 同步策略

建议采用三种同步模式：

- 启动同步
  应用打开时拉取今日关键数据
- 前台增量同步
  每隔一段时间同步邮件与日历摘要
- 手动强制同步
  用户点击刷新时立即同步

推荐策略：

- 首页只读取今日和本周的数据
- 邮件先同步头部信息和摘要，再按需展开正文
- 日历先同步最近 7 天和未来 14 天

### 8.3 错误与状态设计

系统应能显示：

- 已连接
- 同步中
- 授权失效
- 服务暂不可用
- 最近同步时间

### 8.4 隐私与权限

- 邮件正文和日历内容默认只用于本地工作台理解
- 所有接入都应有单独授权说明
- 用户可关闭自动分类和智能建议
- 重要数据应支持本地清除和断开连接

## 9. 桌面端能力建议

桌面应用相比 Web 版更适合做这些能力：

- 系统通知
- 菜单栏快速入口
- 本地日历读取
- 开机启动
- 全局快捷键唤起
- 离线缓存与最近数据可用
- 本地文件快速打开

技术路线建议：

- 桌面容器：Electron 或 Tauri
- UI：React 或原型阶段先用静态页面
- 本地数据：SQLite 或 IndexedDB
- 同步层：统一同步服务，屏蔽不同来源差异

## 10. MVP 范围建议

第一版建议控制在以下范围：

- 首页概览
- 日程卡片与时间轴
- 日程总览入口
- 工作区列表与新建工作区
- 三类工作区模板：代码、图片处理、写作
- 天气驱动背景
- 模拟收件与日历摘要
- 工作区自动归类规则的基础版

建议优先接入顺序：

1. 天气 API
2. Google Calendar
3. Gmail
4. Outlook / Graph
5. IMAP
6. 系统日历本地桥接

先不做：

- 多账号复杂同步
- 自动写回系统日历
- 完整聊天式助手
- 复杂团队协作权限

## 12. 当前版本建议结论

当前最适合的落地策略是：

- 首页做天气驱动的工作现场
- 日历做第一优先级接入源
- 邮件做第二优先级接入源
- 工作区以代码、图片处理、写作为核心模板
- 智能层负责归类、建议和冲突识别

## 13. 总结

基于你现在的两个起点，这个智能工作台最适合发展的方向是：

- 首页做“今天的工作现场”
- 工作区做“上下文容器”
- 邮件和日历做“信息输入源”
- 智能能力做“理解、整理、提醒和推荐”

这样产品会有明确主线，也能从原型逐步演进到真正可用的桌面工作平台。
